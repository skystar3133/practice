# 블로그 스타일 가이드

## 개요
본 문서는 기존 블로그 글들의 말투, 문장 길이, 표현 패턴을 분석하여 작성된 스타일 가이드입니다.

---

## 1. 말투 (Tone of Voice)

### 1.1 기본 톤
- **정중하고 신뢰할 수 있는 톤** 사용
- 높임말을 기본으로 함 (합니다, 했습니다, 하였다)
- 객관적이고 사실 기반의 정보 제공

### 1.2 콘텐츠 타입별 톤

#### 정보 제공 글
```
✓ 산림청에서는 "2002 세계 산의 해"를 기념하고...
✓ 100대 명산에는 국립공원(16), 도립공원(17)...이 선정되었다.
```
- 과거형 사용 (했습니다, 하였다, 되었다)
- 출처와 근거 포함

#### 팁/조언 글
```
✓ 지치기 전에 쉬어야 합니다.
✓ ~하는 것이 좋습니다.
✓ ~해야 합니다.
```
- 명령형/조언형 표현 사용
- 이유와 설명을 함께 제공

#### 리스트/정보 글
```
✓ 1코스(1시간 40분 소요)
✓ 하루재 코스 : 도선사 주차장 - ...
```
- 간결하고 객관적 서술
- 정보를 명확하게 구분

## 2. 문장 길이

### 2.1 문장 길이 기준

| 콘텐츠 타입 | 권장 길이 | 예시 |
|-----------|---------|------|
| 정보 제공 | 30-50글자 | 산의 역사·문화성, 접근성, 선호도, 규모, 생태계 특성 등 5개 항목에 가중치를 부여하였다. |
| 팁/조언 | 25-40글자 | 지치기 전에 쉬어야 합니다. |
| 리스트 | 10-25글자 | 1코스(1시간 40분 소요) |

### 2.2 문장 구성
- 복합문 사용으로 정보 밀도 높임
- 한 문장에 2-3개의 정보 포함 가능
- 마침표(.)로 명확하게 문장 구분

---

## 3. 자주 쓰는 표현

### 3.1 정보 제공 글
```
- "~에 선정되었다"
- "~에는 ~개가"
- "또", "등도"
- "이러한", "이와 같이"
- "배경에는" (이유 설명 시)
```

### 3.2 팁/조언 글
```
- "~해야 합니다"
- "~하는 것이 좋습니다"
- "그러면", "이때"
- "~기 때문이다"
- "~할 수 있습니다"
```

### 3.3 원인-결과 표현
```
- "~하면", "~하므로"
- "따라서", "그러므로"
- "그런데", "하지만"
```

---

## 4. 기호 및 마크업

### 4.1 섹션 구분
- **■** : 개별 팁, 항목 구분
- **♣** : 중요 정보 표시
- **숫자** : 순서 있는 리스트 (1코스, 2코스)

### 4.2 사용 예시
```
■ 산행 중 몸이 지치면...

♣ 산림청에서는 "2002 세계 산의 해"를...

1코스(1시간 40분 소요)
2코스(1시간 40분 소요)
```

---

## 5. 문법 및 맞춤법

### 5.1 주의할 점
- 띄어쓰기 정확하게 (예: "원상회복" X → "원상회복" O)
- 시간 표기: "1시간 40분 소요"
- "~기때문이다" X → "~기 때문이다" O
- "~것이좋습니다" X → "~것이 좋습니다" O

### 5.2 경로 표기법
```
출발지 - 중간지점 - 도착지

예: 도선사 주차장 - 우이대피소 - 인수대피소 - 백운대
```

---

## 6. 구조 및 구성

### 6.1 정보 제공 글 구조
1. 제목 (주제)
2. 배경/소개 (왜?)
3. 세부 내용 (무엇?)
4. 구체적 예시

### 6.2 팁/조언 글 구조
1. 제목 (~하는 방법)
2. **■** 첫 번째 팁 : 상황 + 해결책 + 이유
3. **■** 두 번째 팁 : 상황 + 해결책 + 이유
4. **■** 세 번째 팁 : 상황 + 해결책 + 이유

### 6.3 리스트 글 구조
1. 제목
2. 항목명 + (소요시간 등 메타정보)
3. 상세 경로/설명

## 7. 체크리스트

글을 작성할 때 다음을 확인하세요:

- [ ] 적절한 높임말과 존댓글 사용?
- [ ] 문장 길이가 적절한가? (30-50글자 권장)
- [ ] 정보와 이유가 함께 설명되어 있는가?
- [ ] 띄어쓰기와 맞춤법이 정확한가?
- [ ] 기호(■, ♣)가 일관되게 사용되었는가?
- [ ] 경로/순서가 명확하게 표기되었는가?
- [ ] 복합문으로 정보 밀도를 높였는가?

---

## 8. 참고 사항

- 산행/등산 관련 글이 주 콘텐츠
- 안전 관련 정보는 명령형/권장형으로 작성
- 관광/여행 정보는 객관적이고 간결하게
- 전문 용어 사용 시 일반인도 이해할 수 있도록 설명
