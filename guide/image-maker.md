# 이미지 메이커 에이전트 (Image Maker Agent)

## 개요

본 에이전트는 **draft.md 파일의 이미지 마커를 실제 이미지 파일로 변환**하는 역할을 수행합니다.
- **입력**: draft.md 파일 1개
- **출력**: draft_final.md (이미지 마커가 실제 경로로 치환된 최종 파일)
- **생성물**: images 폴더 안의 PNG 이미지 파일들

---

## 1. 이미지 마커 형식

### 1.1 마커 기본 형식
```html
<!-- IMAGE: [설명] -->
```

### 1.2 마커 예시
```
<!-- IMAGE: 클로드 코드의 인터페이스와 주요 기능을 보여주는 이미지 -->
<!-- IMAGE: 에이전트형 코드 생성 프로세스를 보여주는 플로우 차트 -->
<!-- IMAGE: 다양한 프로그래밍 언어 지원을 나타내는 인포그래픽 -->
```

---

## 2. 작업 순서 (7단계)

### Step 1: 이미지 마커 목록 작성

#### 1.1 draft.md 읽기
- 파일 전체 내용을 읽음
- `<!-- IMAGE: [설명] -->` 패턴 찾기

#### 1.2 마커 목록 작성
- 발견된 모든 마커를 번호순으로 정렬
- 각 마커의 위치(섹션) 기록
- 마커의 설명(대괄호 안의 텍스트) 기록

**작성 예시:**
```
발견된 이미지 마커:
1. [위치] 주제 요약 아래
   [설명] 클로드 코드의 인터페이스와 주요 기능을 보여주는 이미지
   
2. [위치] 본문1 아래
   [설명] 에이전트형 코드 생성 프로세스를 보여주는 플로우 차트

3. [위치] 본문2 아래
   [설명] 다양한 프로그래밍 언어 지원을 나타내는 인포그래픽
```

---

### Step 2: 썸네일 이미지 먼저 생성

#### 2.1 본문 제목 및 분위기 분석
- draft.md의 제목 읽기 (예: "클로드 코드 핵심 기능 3가지")
- 글 전체의 분위기와 톤 파악
- 주요 색상 결정 (기술 블로그 → 파란색/보라색 계열)

#### 2.2 썸네일 생성
- **도구**: Playwright 또는 HTML/CSS를 이용한 캡처
- **크기**: 1200x630px (네이버 블로그 추천 크기)
- **저장 경로**: `images/thumbnail.png`
- **포함 요소**:
  - 글 제목
  - 주요 키워드 (예: "AI", "코딩", "자동화")
  - 시각적으로 돋보이는 배경

**썸네일 예시 요소:**
```
┌─────────────────────────────┐
│   클로드 코드                   │
│   핵심 기능 3가지              │
│                             │
│   [AI] [코딩] [자동화]        │
└─────────────────────────────┘
배경: 그래디언트 (파란색→보라색)
```

---

### Step 3: 각 마커별 템플릿 선택

#### 3.1 템플릿 선택 기준

**Template A: template_scene.html**
- 사용 시기: 장소, 풍경, 분위기를 설명할 때
- 특징: 배경 이미지, 큰 텍스트, 자연스러운 레이아웃
- 예시: "개발 환경의 분위기", "IDE 통합 이미지"

**Template B: template_info.html**
- 사용 시기: 수치, 비교, 인포그래픽이 필요할 때
- 특징: 데이터 시각화, 막대 그래프, 아이콘
- 예시: "30개 언어 지원", "성능 비교 차트"

**Template C: template_step.html**
- 사용 시기: 단계, 순서, 프로세스를 보여줄 때
- 특징: 번호 매김, 화살표, 순차적 레이아웃
- 예시: "코드 생성 프로세스", "MCP 연동 단계"

**Template D: template_quote.html**
- 사용 시기: 인용, 강조 문구, 중요한 정보
- 특징: 큰 폰트, 배경 색상, 따옴표 기호
- 예시: "자동화된 작업에서도 완전한 제어 보장"

**Template E: template_default.html**
- 사용 시기: 분류가 안 되는 모든 경우
- 특징: 일반적인 레이아웃, 텍스트 중심
- 예시: 기본 정보 설명

#### 3.2 마커별 템플릿 선택 예시

```
마커 1: "클로드 코드의 인터페이스와 주요 기능을 보여주는 이미지"
→ Template A (scene) 선택 - 개발 환경의 분위기 표현

마커 2: "에이전트형 코드 생성 프로세스를 보여주는 플로우 차트"
→ Template C (step) 선택 - 순서와 프로세스 표현

마커 3: "다양한 프로그래밍 언어 지원을 나타내는 인포그래픽"
→ Template B (info) 선택 - 데이터 시각화 필요
```

---

### Step 4: Playwright로 이미지 캡처 및 저장

#### 4.1 캡처 환경 설정
- Playwright 브라우저 시작 (Chromium)
- 해상도: 1280x720px (고화질)
- 대기 시간: 2초 (렌더링 완료 대기)

#### 4.2 이미지 저장
- **저장 경로**: `images/img_번호.png`
- **파일명 형식**: `img_01.png`, `img_02.png`, ...
- **형식**: PNG (품질 손실 없음)
- **해상도**: 최소 800x600px 이상

#### 4.3 저장 순서
```
1번 마커 → images/img_01.png 저장
2번 마커 → images/img_02.png 저장
3번 마커 → images/img_03.png 저장
...
```

**저장 확인:**
- 파일이 실제로 존재하는가?
- 파일 크기가 0KB가 아닌가?
- 확장자가 정확한가?

---

### Step 5: 이미지 검수 (필수)

#### 5.1 검수 기준 (4가지 필수 확인)

**✓ 기준 1: 텍스트 절단 확인**
- 텍스트가 이미지 경계에서 잘리지 않았는가?
- 모든 글자가 완전하게 보이는가?
- 줄바꿈이 자연스러운가?

**✓ 기준 2: 색상 대비 확인**
- 배경과 텍스트 색이 겹쳐서 안 읽히는 곳은 없는가?
- 충분한 명도 차이(contrast)가 있는가?
- 색맹/약색맹인 사람도 구분할 수 있는가?

**✓ 기준 3: 내용 정확성 확인**
- 마커 설명과 이미지 내용이 일치하는가?
- 필요한 정보가 모두 포함되었는가?
- 잘못된 정보나 누락은 없는가?

**✓ 기준 4: 해상도 확인**
- 이미지가 흐릿하지 않은가?
- 글자가 명확하게 읽히는가?
- 최소 800x600px 해상도를 충족하는가?

#### 5.2 검수 진행 방식

```
for 각 생성된 이미지 in images 폴더:
    1. 이미지 뷰어로 열기
    2. 4가지 기준 체크리스트 확인
    3. 문제 있는가?
        ⊙ 있으면 → Step 6 (재생성)
        ⊙ 없으면 → Step 7 (마커 치환)
```

#### 5.3 검수 결과 기록

**검수 통과:**
```
[✅ PASS] img_01.png
- 텍스트 절단: 없음
- 색상 대비: 양호
- 내용 정확성: 일치
- 해상도: 1280x720px
```

**검수 실패:**
```
[❌ FAIL] img_02.png
- 텍스트 절단: 오른쪽 끝이 잘림
- 색상 대비: 양호
- 내용 정확성: 일치
- 해상도: 1280x720px
[사유] 텍스트 오버플로우, 재생성 필요
```

---

### Step 6: 문제 있으면 재생성 (반복)

#### 6.1 재생성 필요 시 절차
1. 문제 원인 파악 (텍스트 길이, 색상, 레이아웃 등)
2. 템플릿 수정 또는 다른 템플릿으로 변경
3. Step 4부터 다시 실행 (캡처 → 저장)
4. Step 5에서 재검수

#### 6.2 재생성 기록

```
[재생성 1차]
- 원인: 텍스트가 이미지 오른쪽에서 잘림
- 해결: template_info.html에서 font-size 감소 (18px → 16px)
- 결과: 통과 ✅

[재생성 2차]
- 원인: 배경색과 텍스트 색 대비 부족
- 해결: 배경색 변경 (light gray → dark blue)
- 결과: 통과 ✅
```

#### 6.3 최대 재생성 횟수
- 같은 마커에 대해 최대 3회까지 재생성 허용
- 3회 이후에도 불가능하면 대체 템플릿 선택
- 마지막 수단으로 template_default.html 사용

---

### Step 7: 검수 통과 후 마커 치환

#### 7.1 마커 → 경로 치환

**치환 전:**
```markdown
<!-- IMAGE: 클로드 코드의 인터페이스와 주요 기능을 보여주는 이미지 -->
```

**치환 후:**
```markdown
![클로드 코드의 인터페이스와 주요 기능을 보여주는 이미지](./images/img_01.png)
```

#### 7.2 치환 형식
- **마크다운 이미지 링크**: `![alt텍스트](경로)`
- **alt텍스트**: 마커에 있던 설명 그대로 사용
- **경로**: `./images/img_번호.png` (상대경로)

#### 7.3 검증
- 모든 마커가 치환되었는가?
- 경로가 모두 정확한가?
- 오탈자는 없는가?

---

### Step 8: 최종 파일 저장

#### 8.1 저장 파일명
- **파일명**: `draft_final.md`
- **저장 경로**: 원본 draft.md와 동일한 폴더 (output 폴더)
- **인코딩**: UTF-8

#### 8.2 원본 draft.md 보호
```
⚠️ 반드시 확인:
- draft.md 원본을 수정하지 않았는가?
- draft_final.md만 새로 생성했는가?
- 원본이 그대로 남아있는가?
```

## 3. 템플릿 상세 가이드

### 3.1 Template_scene.html
**용도**: 장소, 풍경, 분위기, 개발 환경

**특징:**
- 배경 이미지 또는 그래디언트
- 큰 텍스트 (36px 이상)
- 자연스러운 공간 활용

**마커 예시:**
- "개발 환경의 분위기 표현"
- "IDE 통합 화면"
- "터미널 작업 환경"

### 3.2 Template_info.html
**용도**: 수치, 통계, 비교, 인포그래픽

**특징:**
- 데이터 시각화 (차트, 그래프)
- 숫자와 단위 강조
- 아이콘 사용

**마커 예시:**
- "30개 언어 지원"
- "성능 비교 차트"
- "기능 비교표"

### 3.3 Template_step.html
**용도**: 단계, 순서, 프로세스, 플로우

**특징:**
- 번호 매김 (1, 2, 3, ...)
- 화살표 (→ 또는 ↓)
- 순차적 배치

**마커 예시:**
- "코드 생성 프로세스"
- "MCP 연동 단계"
- "워크플로우 흐름"

### 3.4 Template_quote.html
**용도**: 인용문, 강조 문구, 중요 정보

**특징:**
- 큰 폰트 (40px 이상)
- 배경색 강조
- 따옴표 기호 또는 옆줄

**마커 예시:**
- "자동화된 작업에서도 완전한 제어 보장"
- "30개 이상의 프로그래밍 언어 지원"
- "중요한 기능 강조"

### 3.5 Template_default.html
**용도**: 분류되지 않는 모든 경우

**특징:**
- 일반적인 레이아웃
- 텍스트 중심
- 무난한 디자인

**마커 예시:**
- 어느 템플릿도 어울리지 않는 설명
- 복합적인 정보

## 4. 검수 체크리스트

### 4.1 개별 이미지 검수 체크리스트

```
이미지 파일명: img_00.png
마커 설명: [여기에 마커 설명 입력]

[ ] 기준 1: 텍스트 절단 확인
    - 텍스트가 완전한가?
    - 줄바꿈이 자연스러운가?
    - 경계 여백이 충분한가?

[ ] 기준 2: 색상 대비 확인
    - 배경과 텍스트가 구분되는가?
    - 명도 차이가 충분한가?
    - 색맹/약색맹도 구분 가능한가?

[ ] 기준 3: 내용 정확성 확인
    - 마커와 내용이 일치하는가?
    - 필요한 정보가 모두 있는가?
    - 오류가 없는가?

[ ] 기준 4: 해상도 확인
    - 흐릿하지 않은가?
    - 글자가 명확한가?
    - 최소 800x600px인가?

[ ] 최종 결정
    ⊙ 검수 통과 (PASS)
    ⊙ 검수 실패 (FAIL) - 사유: ___________
```

### 4.2 전체 작업 검수 체크리스트

```
[ ] Step 1: 마커 목록 작성 완료
[ ] Step 2: 썸네일 이미지 생성 (images/thumbnail.png)
[ ] Step 3: 모든 마커에 템플릿 선택 완료
[ ] Step 4: 모든 이미지 파일 저장 확인
[ ] Step 5: 모든 이미지 검수 통과
[ ] Step 6: 재생성 필요한 이미지 처리 완료
[ ] Step 7: 모든 마커 경로로 치환 완료
[ ] Step 8: draft_final.md 저장 완료
[ ] Step 9: 원본 draft.md가 수정되지 않았음 확인
```

## 5. 금지사항 (⚠️ 필수 준수)

### 5.1 검수 생략 금지
```
❌ 금지: 검수하지 않고 바로 마커를 경로로 치환
✅ 필수: Step 5에서 4가지 기준을 모두 확인 후 치환
```

**위반 시 결과:**
- 텍스트가 잘린 이미지 게시
- 색상 대비가 안 맞는 이미지 게시
- 내용과 다른 이미지 게시
- 흐릿한 이미지 게시

### 5.2 draft.md 원본 수정 금지
```
❌ 금지: 원본 draft.md 파일 직접 수정
✅ 필수: draft_final.md 새 파일로 저장
```

**원본 보호 절차:**
1. draft.md 읽기 (수정 금지)
2. 내용을 메모리에 저장
3. draft_final.md에 수정 내용 저장
4. 원본 draft.md는 그대로 유지

### 5.3 이미지 저장 필수
```
❌ 금지: 이미지를 실제로 저장하지 않고 경로만 입력
✅ 필수: images 폴더에 PNG 파일로 실제 저장
```

**저장 확인 방법:**
```
파일 탐색기에서 확인:
D:\workspace\blog\output\images\
├── thumbnail.png
├── img_01.png
├── img_02.png
└── img_03.png
```

## 6. 완료 보고 형식

### 6.1 최종 보고

작업 완료 후 다음 형식으로 보고합니다.

```markdown
## 📊 이미지 생성 완료 보고

### 1. 처리한 마커 수
- **총 마커 개수**: 3개
- **생성된 이미지**: 3개
- **썸네일**: 1개
- **전체 생성 이미지**: 4개

### 2. 생성된 이미지 목록
```
