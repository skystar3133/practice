# 블로그 라이터 지침서 (Writer's Guidelines)

## 개요

본 지침서는 **세 가지 가이드를 모두 반영**하여 블로그 글을 작성하는 규칙입니다.
- **Research-Result.md**: 정보 내용 및 주제 선정
- **Style-Guide.md**: 말투, 문장 길이, 표현 방식
- **SEO-Guide.md**: 제목, 구조, 키워드 배치

라이터는 이 세 가이드를 통합하여 검색 최적화되고, 읽기 쉽고, 신뢰할 수 있는 글을 작성합니다.

---

## 1. 글 작성 전 준비

### 1.1 리서치 자료 확보
- **Research-Result.md 확인**: 해당 주제의 리서치 결과 검토
- 각 기능/항목마다 제시된 10-15개의 신뢰할 수 있는 출처 확인
- 핵심 내용, 세부 설명, 구체적 예시 파악

### 1.2 주제 선정
- 리서치 결과의 10가지 핵심 항목 중 선택 (예: "클로드 코드의 핵심 기능 5가지")
- 또는 개별 기능 심화 글 작성 (예: "MCP 서버 연동 완벽 가이드")
- 독자 수준과 관심도 고려

## 2. 제목 작성 규칙 (SEO-Guide 기반)

### 2.1 제목 길이 및 키워드
```
✓ 20자 이내 작성 (필수)
✓ 메인 키워드를 제목 앞쪽(처음 10자 이내)에 배치
✓ 숫자나 기호 포함하면 클릭률 ↑
```

### 2.2 제목 작성 예시
```
좋은 예:
- 클로드 코드 핵심 기능 5가지
- AI 코드 생성 자동화 완벽 가이드
- MCP 서버 연동 3단계

피해야 할 예:
- 클로드 코드가 정말 좋은 이유를 알려드릴게요 (길음)
- 개발자를 위한 도구 (키워드 불명확)
```

### 2.3 키워드 배치
- **제목**: 메인 키워드 (예: "클로드 코드", "AI 코드 생성")
- **주제 요약**: 메인 키워드 포함
- **소제목**: 세부 키워드 (예: "MCP 서버", "IDE 통합")

## 3. 본문 구조 (SEO-Guide 6단계)

### 3.1 필수 구성 순서

```
1️⃣ 주제 요약 (50-100자, 2-3줄)
   ↓
2️⃣ 이미지 (1개 필수, 800x600px 이상)
   ↓
3️⃣ 본문1 (첫 번째 핵심 내용, 150-200자)
   ↓
4️⃣ 본문2 (두 번째 핵심 내용, 150-200자)
   ↓
5️⃣ 본문3 (세 번째 핵심 내용, 150-200자)
   ↓
6️⃣ 마무리 (결론 및 행동 유도, 50-100자)
```

### 3.2 각 단계별 작성 방법

#### 1단계: 주제 요약
- **목적**: 글의 주제와 핵심을 한눈에 파악
- **길이**: 50-100자 (2-3줄)
- **필수 포함**: 메인 키워드
- **톤**: 정중한 높임말 (Style-Guide)
- **표현**: "이 글에서는...", "본 글은..."

**예시:**
```
이 글에서는 클로드 코드의 5가지 핵심 기능을 소개합니다.
개발 생산성을 획기적으로 높일 수 있는 기능들을 상세하게 설명하겠습니다.
```

#### 2단계: 이미지
- **개수**: 정확히 1개 (필수)
- **크기**: 최소 800x600px 이상, 권장 1200x800px
- **내용**: 글의 주제와 관련 있는 고화질 이미지
- **위치**: 주제 요약 아래, 본문1 위
- **설명**: 이미지 아래 간단한 캡션 추가

#### 3-5단계: 본문1, 본문2, 본문3
- **구조**: 각각 독립적인 주제
- **길이**: 150-200자 (최소 100자)
- **소제목**: ■ 기호로 시작 (Style-Guide)
- **내용**: Research-Result.md의 정보 활용
  - 핵심 내용 (한 문장 정의)
  - 세부 설명 (3-5문장)
  - 구체적 예시 (실제 사용 사례)

**구조 예시:**
```
■ 본문1: 기능명 또는 핵심 키워드

핵심 내용을 한 문장으로 정의합니다.

세부 설명을 이해하기 쉽게 풀어서 작성합니다. 
복합문을 사용하여 정보 밀도를 높입니다. 
전문 용어는 괄호로 설명을 추가합니다.

구체적 예시:
- 첫 번째 사용 사례
- 두 번째 사용 사례
- 세 번째 사용 사례
```

#### 6단계: 마무리
- **길이**: 50-100자 (2-3줄)
- **내용**: 결론 또는 요약
- **행동 유도**: 댓글, 공감, 구독 권장
- **톤**: 정중하고 친근하게

**예시:**
```
클로드 코드의 이 다섯 가지 기능을 활용하면 개발 효율이 크게 높아집니다.
지금 바로 사용해보고 여러분의 경험을 댓글로 공유해주세요.
```

## 4. 말투 및 표현 규칙 (Style-Guide 기반)

### 4.1 기본 톤
```
✓ 정중하고 신뢰할 수 있는 톤
✓ 높임말 기본 (합니다, 했습니다, 하였다)
✓ 객관적이고 사실 기반의 정보 제공
```

### 4.2 문장 길이
```
정보 제공 글: 30-50글자
예: "클로드 코드는 AI를 통해 자동으로 코드를 생성하고 수정합니다."

팁/조언 글: 25-40글자
예: "이 기능을 활용하면 개발 시간을 단축할 수 있습니다."
```

### 4.3 자주 쓰는 표현 (정보 제공)

**추천 표현:**
```
- "~에 최적화되었다"
- "~는 특징이 있습니다"
- "~를 지원합니다"
- "이러한", "이와 같이"
- "또한", "더불어"
- "그 결과", "따라서"
```

**피해야 할 표현:**
```
✗ "정말 좋아요", "꼭 사용해야 해요" (주관적)
✗ "절대 빠뜨리면 안 됩니다" (과장)
✗ "~기 때문인데요" (구어체)
```

### 4.4 기호 사용 (Style-Guide)
```
■ (정사각형 검은색 기호)
  - 개별 팁이나 항목 구분
  - 본문1, 본문2, 본문3 소제목 앞

♣ (클럽 기호)
  - 매우 중요한 정보 강조
  - 매 글마다 1-2개 이하로 사용

- (하이픈)
  - 불릿 포인트 사용
  - 리스트 항목 구분
```

**사용 예시:**
```
■ 첫 번째 기능: 에이전트형 AI 코드 생성

♣ 중요: 이 기능은 전체 코드베이스를 이해하고 작업합니다.

- 구체적 예시 1
- 구체적 예시 2
```

## 5. 키워드 배치 전략 (SEO-Guide)

### 5.1 키워드 배치 위치
```
1순위: 제목 (가장 중요)
2순위: 주제 요약 (첫 단락)
3순위: 소제목 (본문1, 2, 3)
4순위: 본문 중간 (자연스럽게 2-3회)
```

### 5.2 메인 키워드 사용 빈도
```
✓ 메인 키워드 3-5회 자연스럽게 분산
✗ 같은 단어 5회 이상 반복 (스팸 판정)
✓ 연관 키워드와 함께 사용 (동의어 활용)
```

**예시 - "클로드 코드" 글:**
```
제목: 클로드 코드 핵심 기능 5가지
주제 요약: 클로드 코드는 AI 기반의 코딩 도구입니다.
본문1 소제목: ■ 클로드 코드의 첫 번째 기능
본문2: 이 도구가 다른 코드 생성 도구와 다른 이유는...
본문3: 클로드 코드를 사용하면...
```

## 6. Research-Result 정보 활용법

### 6.1 정보 선택
- **핵심 내용**: 한 문장으로 정의 → 본문 소제목이나 첫 문장
- **세부 설명**: 3-5문장 → 본문의 중심 내용
- **구체적 예시**: 실제 사례 → 본문 끝부분

### 6.2 정보 변환
```
원문 (Research-Result):
"클로드 코드는 개발자가 자연언어로 원하는 기능을 설명하면, 
전체 코드베이스를 읽고 계획을 세워 여러 파일에 걸쳐 
자동으로 코드를 작성하고 수정합니다."

라이터 버전 (Style-Guide 적용):
"자연언어로 원하고자 하는 기능을 설명하기만 하면, 
클로드 코드가 전체 코드베이스를 파악하고 
자동으로 코드를 작성합니다."
```

### 6.3 출처 표기
- **출처 위치**: 각 본문 섹션 아래
- **형식**: [출처명] - URL
- **신뢰도**: 모든 정보가 신뢰할 수 있는 출처에서 나온 것

## 7. 작성 순서 및 체크리스트

### 7.1 작성 순서
```
1단계: 제목 작성
   → SEO-Guide 제목 규칙 확인 (20자 이내, 키워드 앞쪽)

2단계: 주제 요약 작성
   → Research-Result의 개요 참고
   → 50-100자, 높임말 사용

3단계: 이미지 준비
   → 관련 고화질 이미지 확보 (800x600px 이상)

4단계: 본문1, 2, 3 작성
   → Research-Result의 각 기능 순서대로
   → Style-Guide의 문장 길이 준수 (30-50글자)
   → 기호(■) 사용, 예시 포함

5단계: 마무리 작성
   → 결론 + 행동 유도
   → 독자 댓글 권장

6단계: 전체 검토
   → 아래 체크리스트 확인
```

### 7.2 최종 체크리스트

**제목 및 키워드:**
- [ ] 제목이 20자 이내인가?
- [ ] 메인 키워드가 제목 앞쪽(10자 이내)에 있는가?
- [ ] 제목에 숫자나 기호가 포함되었는가?
- [ ] 메인 키워드가 3-5회 자연스럽게 분산되었는가?

**말투 및 표현:**
- [ ] 정중한 높임말을 사용했는가? (합니다, 했습니다)
- [ ] 객관적이고 사실 기반인가?
- [ ] 문장 길이가 30-50글자인가? (정보 제공 글)
- [ ] 복합문으로 정보 밀도를 높였는가?

**본문 구조 (6단계):**
- [ ] 주제 요약이 명확한가? (50-100자, 2-3줄)
- [ ] 이미지가 1개 정확히 포함되었는가?
- [ ] 이미지 크기가 800x600px 이상인가?
- [ ] 본문1, 2, 3이 각각 150-200자 이상인가?
- [ ] 소제목에 ■ 기호가 사용되었는가?
- [ ] 마무리 문장이 있는가? (50-100자)

**기호 및 표현:**
- [ ] ■ 기호가 각 본문 소제목에 사용되었는가?
- [ ] 중요 정보는 ♣ 또는 **굵게** 표시했는가?
- [ ] 불릿 포인트(-)로 예시를 구분했는가?

**리서치 및 출처:**
- [ ] Research-Result의 정보를 올바르게 활용했는가?
- [ ] 모든 주요 정보에 출처가 명시되었는가?
- [ ] 출처 URL이 정확한가?

**기타:**
- [ ] 맞춤법과 띄어쓰기가 정확한가?
- [ ] 전문 용어에 설명이 있는가?
- [ ] 독자에게 행동 유도(댓글, 공감)가 있는가?
- [ ] 전체 본문이 600자 이상인가?

```
# 클로드 코드 핵심 기능 5가지

클로드 코드는 AI 기반의 에이전트형 코딩 도구입니다. 
단순한 코드 편집을 넘어 개발 전체 과정을 효율화합니다.
이 글에서는 반드시 알아야 할 5가지 핵심 기능을 소개합니다.

[이미지: 클로드 코드 인터페이스]

■ 기능 1: AI 기반 코드 생성

자동으로 코드를 작성합니다. 전체 코드베이스를 이해하고 
여러 파일을 동시에 수정할 수 있다는 것이 특징입니다.

예시:
- "로그인 기능을 구현해줘" → 인증 로직 + DB 연결 + 테스트 코드 자동 생성

■ 기능 2: 지능형 코드 리뷰

작성한 코드의 버그, 보안 문제, 성능 이슈를 자동으로 검출합니다.
각 지적마다 이유와 개선 방법을 설명해주므로 
코드 품질을 높일 수 있습니다.

...

---

마무리

클로드 코드의 이 다섯 가지 기능을 활용하면 
개발 효율이 크게 높아집니다.
지금 바로 경험해보고 댓글로 여러분의 경험을 공유해주세요.
```

## 9. 주의사항

### 9.1 피해야 할 것
```
✗ Research-Result 내용의 무단 복사
   → 자신의 말로 정리하고 출처만 명시

✗ 과도한 전문 용어 사용
   → 모든 용어에 괄호로 설명 추가

✗ 주관적인 표현
   → "정말 좋습니다" 대신 "효율이 높습니다" 사용

✗ 부정확한 정보
   → Research-Result의 신뢰 가능한 출처만 참고

✗ SEO 스팸 행위
   → 키워드 무한 반복, 숨겨진 텍스트 금지
```

### 9.2 품질 기준
```
✓ 최소 3개 이상의 신뢰할 수 있는 출처 인용
✓ 모든 주요 주장에 근거 제시
✓ 읽기 쉬운 문장 (30-50글자)
✓ 명확하고 체계적인 구조
✓ 정중한 높임말 유지
```

## 10. 예상 작성 시간 및 분량

### 10.1 예상 분량
- **제목**: 20자 이내 (1줄)
- **주제 요약**: 50-100자 (2-3줄)
- **이미지**: 1개
- **본문1, 2, 3**: 150-200자 × 3 = 450-600자
- **마무리**: 50-100자 (2-3줄)
- **전체**: 600-800자

### 10.2 예상 작성 시간
- Research-Result 검토: 10-15분
- 제목 및 주제 요약: 5-10분
- 본문 작성: 30-45분
- 이미지 준비: 5-10분
- 검토 및 수정: 10-15분
- **총 소요 시간: 60-95분**

## 최종 요약

**라이터는 이 세 가지를 동시에 만족해야 합니다:**

1. **Research-Result 기반**: 신뢰할 수 있는 정보만 활용
2. **Style-Guide 기반**: 정중한 높임말, 30-50글자 문장, 기호 사용
