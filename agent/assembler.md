# 어셈블러 (Assembler) 지침서

## 개요

어셈블러는 완성된 블로그 글(draft.md)과 생성된 이미지들을 조합하여 최종 게시 가능한 형태의 파일을 만드는 도구입니다. 
블로그 글의 이미지 경로가 올바르게 참조되고, 모든 리소스가 준비된 완전한 형태의 최종 산출물을 생성합니다.

---

## 1. 입력 파일

### 1.1 주요 입력
```
입력 1. 블로그 글 파일
  파일명: draft.md (또는 draft_final.md)
  위치: D:\workspace\blog\output\draft.md
  상태: 완성된 글 + 이미지 경로 포함 [이미지] 마커 또는 ![설명](./images/img_XX.png) 형식

입력 2. 이미지 파일들
  폴더: D:\workspace\blog\output\images\
  구성:
    - thumbnail.png (1200x630px) - Naver 블로그 썸네일
    - img_01.png (1280x720px) - 메인 콘텐츠 이미지
    - img_02.png (1280x720px) - 2번째 섹션 이미지
    - img_03.png (1280x720px) - 3번째 섹션 이미지
    - img_04.png (1280x720px) - 4번째 섹션 이미지
```

### 1.2 입력 검증 기준
```
✓ draft.md 존재 확인
✓ images 폴더 존재 확인
✓ 모든 이미지 파일(img_01~img_04, thumbnail) 존재 확인
✓ draft.md 내 이미지 경로가 올바른 형식인지 확인
  - 형식 1: [이미지] (마커)
  - 형식 2: ![설명](./images/img_XX.png) (실제 경로)
✓ draft.md 파일 크기 확인 (최소 500바이트 이상)
```

## 2. 어셈블리 프로세스

### Step 1: 상태 분석
```
작업 내용:
  1) draft.md 파일 읽기
  2) 이미지 경로 형식 분석
  3) 이미지 마커 위치 파악
  4) images 폴더 파일 목록 확인

검사 항목:
  - 마커 형식: [이미지] 또는 ![](./images/img_XX.png)?
  - 마커 개수: 몇 개인가?
  - 이미지 파일: 모두 있는가?
```

### Step 2: 경로 정규화
```
목적: 모든 이미지 경로를 통일된 형식으로 변환

규칙:
  입력 형식 1: [이미지]
    → 해당 위치에 첫 번째 이미지부터 차례로 매칭
    → 마커 1 → img_01.png
    → 마커 2 → img_02.png (있을 경우)

  입력 형식 2: ![설명](./images/img_XX.png)
    → 그대로 유지 (이미 정규화됨)

  잘못된 형식: ![설명](./img_XX.png), ![설명](images/img_XX.png)
    → ./images/ 경로로 정정

최종 형식:
  ![이미지 설명](./images/img_01.png)
  ![이미지 설명](./images/img_02.png)
  ...
```

### Step 3: 마크다운 검증
```
검사 항목:
  1) 제목 존재 여부 (# 마크다운 형식)
  2) 이미지 경로 문법 올바름
     - ![](./images/...) 형식 확인
     - 상대 경로 정확성 확인
  3) 링크 문법 올바름
     - [텍스트](URL) 형식
  4) 마크다운 기본 구조
     - 섹션 구분 (---)
     - 리스트 형식 (-, *)
     - 강조 형식 (**굵게**, *이탤릭*)

수정 필요 시:
  - 문법 오류 수정
  - 경로 정정
  - 형식 통일
```

### Step 4: 파일 통합
```
작업 내용:
  1) draft.md 내용 그대로 유지
  2) 모든 이미지 경로가 ./images/img_XX.png 형식 확인
  3) thumbnail.png는 별도 메타데이터로 기록

결과 파일 생성:
  - 파일명: draft_final.md (또는 project_final.md)
  - 저장 위치: D:\workspace\blog\output\
  - 내용: 완성된 마크다운 + 정규화된 이미지 경로
```

### Step 5: 최종 검수
```
검사 기준 (4가지):

1) 이미지 경로 정확성
   ✓ 모든 ![](./images/...) 경로가 실제 파일과 매칭
   ✓ 상대 경로가 output/ 디렉토리 기준인가?
   ✓ 경로에 띄어쓰기나 특수문자 없음

2) 마크다운 문법 완전성
   ✓ 모든 ![...] 형식이 올바른 문법
   ✓ 제목, 리스트, 강조 형식 올바름
   ✓ 파일에 인코딩 문제 없음

3) 콘텐츠 무결성
   ✓ draft.md 원본 내용 전부 포함
   ✓ 이미지 마커가 실제 경로로 완전히 치환
   ✓ 순서 변경 없음

4) 파일 완성도
   ✓ 파일 크기가 원본과 유사 (일부 증가 가능)
   ✓ 저장 형식: UTF-8 인코딩
   ✓ 줄바꿈 형식: LF (또는 CRLF 일관성)
```

### Step 6: 재생성 판단
```
PASS 기준 (재생성 불필요):
  ✓ 4가지 검수 기준 모두 통과

FAIL 기준 (재생성 필요):
  ✗ 이미지 경로 오류 → 경로 수정 후 재생성
  ✗ 마크다운 문법 오류 → 문법 수정 후 재생성
  ✗ 이미지 파일 누락 → image-maker 재실행 필요
  ✗ 인코딩 문제 → UTF-8로 인코딩 변환

재생성 시:
  - 오류 원인 명확히 파악
  - 해당 부분만 수정
  - 5단계부터 다시 검수
  - 최대 3회 반복 (3회 이상 필요 시 image-maker 확인)
```

### Step 7: 최종 통합
```
최종 산출물 준비:
  1) draft_final.md 파일 확정
  2) images 폴더 파일 확인
  3) 메타데이터 생성

메타데이터:
  - 썸네일: thumbnail.png (1200x630px)
  - 본문 이미지 4개: img_01~img_04 (1280x720px)
  - 총 파일 크기
  - 생성 완료 시간
```

### Step 8: 게시 준비
```
최종 확인 체크리스트:
  [ ] draft_final.md 생성 완료
  [ ] 모든 이미지 파일 output/images/ 내 존재
  [ ] 이미지 경로 ./images/img_XX.png 형식 통일
  [ ] 마크다운 문법 오류 없음
  [ ] UTF-8 인코딩 확인
  [ ] 파일 크기 정상 (500바이트 이상)
  
게시 가능 상태:
  ✓ draft_final.md + images 폴더 → Naver 블로그 업로드 준비 완료
```

## 3. 검수 기준 (자세히)

### 3.1 이미지 경로 정확성

**확인 사항:**
```
형식: ![대체 텍스트](./images/img_XX.png)

✓ 경로는 ./images/로 시작
✓ 파일명은 img_01.png, img_02.png 등 숫자 형식
✓ 파일 확장자는 .png
✓ 경로에 공백이나 특수문자 없음
✓ 상대 경로는 output/ 기준

❌ 틀린 예:
  - ![](./img_01.png) → images 폴더 누락
  - ![](images/img_01.png) → ./ 누락
  - ![](D:/workspace/blog/output/images/img_01.png) → 절대 경로 사용
  - ![](./images/img 01.png) → 파일명 띄어쓰기
```

### 3.2 마크다운 문법 완전성

**확인 사항:**
```
마크다운 요소별 검사:

제목 (Heading):
  ✓ # 로 시작, 한 칸 띄우기
  ✓ 예: # 제목 (O), #제목 (X)

이미지 (Image):
  ✓ ![텍스트](경로) 형식
  ✓ 대괄호와 소괄호 쌍 맞기
  ✓ 경로 올바름

링크 (Link):
  ✓ [텍스트](URL) 형식
  ✓ URL이 유효한 형식
  ✓ 대괄호와 소괄호 쌍 맞기

리스트 (List):
  ✓ - 또는 * 일관성
  ✓ 각 항목 - 다음 한 칸 띄우기
  ✓ 중첩 시 들여쓰기 일관성 (2칸)

강조 (Emphasis):
  ✓ **굵게** (O), ***굵고 기울임*** (O)
  ✓ *기울임* (O)

구분선 (Horizontal Rule):
  ✓ --- (3개 이상 대시)
  ✓ *** (3개 이상 별표)
```

### 3.3 콘텐츠 무결성

**확인 사항:**
```
내용 검증:

✓ 원본 draft.md의 모든 텍스트 포함
✓ 문단 순서 변경 없음
✓ 출처 링크 모두 유지
✓ 이미지 마커가 실제 경로로 완전히 치환
✓ 추가 또는 삭제된 내용 없음

변수 확인:
  - 단어 개수: 원본과 동일
  - 문단 수: 원본과 동일
  - 이미지 개수: 원본 마커 수와 일치
```

### 3.4 파일 완성도

**확인 사항:**
```
파일 속성:

✓ 인코딩: UTF-8 (BOM 없음)
✓ 줄바꿈: LF 또는 CRLF 일관성
✓ 파일 크기: 원본보다 크거나 같음
  (경로가 추가되므로 보통 원본보다 큼)
✓ 마지막 문자: 줄바꿈(\n)으로 끝남

예상 크기:
  - draft.md: ~7KB
  - draft_final.md: ~8-10KB
  (이미지 경로가 추가되므로 약간 증가)
```

## 4. 출력 파일 명명 규칙

### 4.1 기본 파일명
```
base_name: 프로젝트명 또는 글 주제
예: claude-code (클로드 코드)

최종 파일명 형식:
  - {base_name}_final.md
  - 예: claude-code_final.md

또는

  - draft_final.md (기본값)
```

### 4.2 저장 위치
```
기본 저장 경로:
  D:\workspace\blog\output\{final_filename}.md

예시:
  D:\workspace\blog\output\draft_final.md
  또는
  D:\workspace\blog\output\claude-code_final.md
```

### 4.3 버전 관리
```
재생성 시 파일명 규칙:
  - 1회차: draft_final.md
  - 2회차: draft_final_v2.md (필요 시)
  - 3회차: draft_final_v3.md (필요 시)

권장사항:
  - 보통 1회차에서 완성
  - 오류 발생 시에만 버전 증가
  - 최종 결과는 항상 _final.md로 명명
```

## 7. 사용 사례

### 사용 사례 1: 기본 어셈블리
```
상황: draft.md와 4개의 이미지가 준비됨

Step 1: draft.md 읽음 → 1개의 [이미지] 마커 발견
Step 2: 마커를 ![설명](./images/img_01.png) 형식으로 변환
Step 3-4: 마크다운 검증 및 파일 통합
Step 5: 4가지 검수 기준 모두 통과
결과: draft_final.md 생성 완료 (게시 가능)
```

### 사용 사례 2: 이미지 경로 수정 필요
```
상황: draft.md에 경로 오류 있음 (./img_01.png)

Step 1-2: 경로 오류 발견
Step 3: 경로 수정 (./images/img_01.png)
Step 4: 수정된 내용으로 재통합
Step 5: 재검수 (통과)
결과: draft_final.md 생성 완료
```

### 사용 사례 3: 이미지 파일 누락
```
상황: img_03.png 파일이 없음

Step 1: 파일 누락 감지
결과: image-maker 다시 실행 필요
조치: img_03.png 생성 후 어셈블리 재시작
```

## 8. 트러블슈팅

### 문제 1: 이미지 경로 오류
```
증상: 생성된 파일에서 이미지가 표시되지 않음

원인 확인:
  1) 경로 형식이 ./images/img_XX.png인가?
  2) 파일명이 정확한가?
  3) 파일이 실제로 존재하는가?

해결:
  1) 경로 정규화 다시 실행
  2) 파일명 확인 및 수정
  3) image-maker로 누락된 이미지 생성
```

### 문제 2: 마크다운 문법 오류
```
증상: 생성된 파일의 마크다운이 렌더링되지 않음

원인 확인:
  1) ![...] 형식이 올바른가?
  2) 대괄호-소괄호 쌍이 맞는가?
  3) 경로에 공백이나 특수문자 있는가?

해결:
  1) 문법 검사 도구 사용
  2) 오류 부분 수정
  3) 재검수
```

### 문제 3: 인코딩 오류
```
증상: 한글이나 특수문자가 깨져 보임

원인: 인코딩이 UTF-8이 아님

해결:
  1) 파일을 UTF-8 인코딩으로 변환
  2) BOM 제거
  3) 파일 저장 후 재확인
```

### 문제 4: 파일 크기 이상
```
증상: draft_final.md 크기가 예상과 다름

확인:
  1) 경로 추가로 크기 증가 정상 (몇KB)
  2) 원본보다 작으면 내용 누락 확인
  3) 너무 크면 중복 내용 확인

해결:
  1) 내용 비교 (원본과 최종)
  2) 중복 제거
  3) 필요시 재생성
```

## 9. 최종 산출물 구조

```
  - 제목 (20자 이내)

```
