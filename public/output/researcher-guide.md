# 리서처 지침서 (Research Guidelines)

## 개요
본 지침서는 블로그 글 작성을 위한 신뢰할 수 있는 정보를 수집하고 정리하는 프로세스입니다.

---

## 1. 정보 수집 규칙

### 1.1 신뢰할 수 있는 출처 선정 기준

#### 우선순위 높은 출처
```
1순위: 공식 문서 및 공식 사이트
   - 제품/서비스 공식 홈페이지
   - 공식 깃허브 저장소
   - 공식 개발 가이드/API 문서
   - 공식 블로그

2순위: 전문가 및 신뢰할 수 있는 기관
   - 해당 분야 전문가 블로그
   - 대학/연구기관 자료
   - 언론사 기술 기사
   - 공인된 교육 플랫폼

3순위: 커뮤니티 자료
   - Stack Overflow
   - Reddit (커뮤니티에서 검증된 답변)
   - GitHub Issues/Discussions
   - 공개된 포럼
```

#### 피해야 할 출처
```
✗ 무명의 개인 블로그 (검증 불가)
✗ 출처 불명확한 기사
✗ 광고성 콘텐츠
✗ 너무 오래된 자료 (5년 이상)
✗ 검수되지 않은 위키 편집
```

### 1.2 정보 검증 프로세스
- 최소 2개 이상의 신뢰할 수 있는 출처에서 같은 정보 확인
- 날짜 확인 (최신 정보인지 확인)
- 출처의 전문성과 신뢰도 평가
- 상충되는 정보는 최신 공식 출처 우선

---

## 2. 정보 수집 범위

### 2.1 수집할 정보 항목
```
✓ 핵심 개념 및 정의
✓ 주요 기능 및 특징
✓ 사용 방법 및 사례
✓ 장점 및 한계
✓ 비교 정보 (유사 제품과의 비교)
✓ 최신 업데이트 및 변경사항
✓ 통계 및 데이터
```

### 2.2 정보 깊이 수준
- **기본 정보**: 초보자도 이해할 수 있도록
- **심화 정보**: 실무자/전문가가 활용할 수 있도록
- **미래 전망**: 제품/서비스의 발전 방향

---

## 3. 정보 정리 방법

### 3.1 정리 구조 (모든 정보는 이 구조로)

```markdown
## [주제]

### 핵심 내용
- 한 문장으로 정의

### 세부 설명
- 이해하기 쉬운 말로 풀어서 설명
- 3-5문장 분량

### 구체적 예시
- 실제 사용 예시나 사례

### 관련 정보
- 연관되는 다른 개념

---
**출처:** 
- [출처명] - https://example.com
- [출처명] - https://example.com
```

### 3.2 라이터 이해도 기준
- 해당 분야 경험 없는 사람이 읽고 이해할 수 있는 수준
- 전문 용어는 괄호로 설명 추가
- 복잡한 개념은 비유나 예시로 설명
- 한국어로 자연스럽게 작성

### 3.3 정보 표현 방식
```
✓ 불릿 포인트 (-) 사용
✓ 숫자로 단계별 설명
✓ 표로 비교 정보 정리
✓ 예시로 구체적 상황 설명
✓ 강조 필요 시 **굵게** 표시
```

## 4. 출처 기록 방법

### 4.1 출처 기록 규칙
```
형식: [출처명] - URL (접근일: YYYY-MM-DD)

예시:
- [Claude 공식 문서] - https://claude.ai/docs (접근일: 2026-07-09)
- [Anthropic 블로그] - https://www.anthropic.com/blog (접근일: 2026-07-09)
```

### 4.2 출처 위치
- 각 섹션 아래 "출처:" 표기
- 글 말미에 전체 출처 목록
- URL은 직접 클릭 가능하도록 링크 형식

---

## 5. 정보 수집 프로세스

### Step 1: 주제 분석
```
- 주제의 범위 정의
- 필요한 정보 항목 나열
- 검색 키워드 결정
```

### Step 2: 웹 검색
```
- 공식 사이트 먼저 검색
- 최신 정보 우선
- 신뢰도 높은 출처 우선 수집
- 최소 5-10개 이상의 자료 확보
```

### Step 3: 정보 검증
```
- 같은 정보가 여러 출처에서 반복되는지 확인
- 모순되는 정보 발견 시 출처 확인
- 날짜 확인 (최신 정보 우선)
```

### Step 4: 정보 정리
```
- 수집한 정보를 위의 구조(3.1)로 정리
- 라이터가 이해할 수 있도록 쉬운 말로 작성
- 출처 URL 정확히 기록
```

### Step 5: 최종 검토
```
- 모든 정보가 신뢰할 수 있는 출처에서 나왔는가?
- 정보가 최신인가?
- 설명이 명확하고 이해하기 쉬운가?
- 출처가 모두 기록되었는가?
```

---

## 6. 검색 키워드 전략

### 6.1 효과적인 검색 키워드
```
기본 키워드: 제품명 + 기본 용어
예: "Claude Code features", "클로드 코드 기능"

심화 키워드: 구체적 기능명
예: "Claude Code AI code generation", "클로드 코드 코드 생성"

비교 키워드: 유사 제품과의 비교
예: "Claude Code vs GitHub Copilot", "클로드 코드 비교"

최신 정보: 연도 포함
예: "Claude Code 2026 updates", "클로드 코드 2026"
```

### 6.2 검색 전략
- 영어 검색과 한글 검색 병행
- 공식 사이트 검색 먼저
- 한글로 된 신뢰할 수 있는 자료도 포함
- 날짜 필터 사용 (최근 1-2년 자료)

---

## 7. 주의사항

### 7.1 피해야 할 행동
```
✗ AI가 생성한 내용만 믿고 인용
✗ 한 출처만 참고하여 정보 수집
✗ 확인되지 않은 정보 포함
✗ 출처 없이 정보 제시
✗ 너무 오래되거나 검증되지 않은 자료 사용
✗ 저작권 문제 있는 내용 직접 인용
```

### 7.2 품질 기준
```
✓ 최소 5개 이상의 신뢰할 수 있는 출처
✓ 모든 주요 정보에 대한 출처 명기
✓ 라이터가 바로 글을 쓸 수 있는 수준
✓ 정보의 통일성과 일관성 유지
✓ 명확하고 이해하기 쉬운 표현
```

## 9. 체크리스트

리서치 완료 전 확인:
- [ ] 최소 5개 이상의 신뢰할 수 있는 출처 확보?
- [ ] 모든 정보가 출처와 함께 기록되었는가?
- [ ] 라이터가 바로 글을 쓸 수 있는 수준인가?
- [ ] 정보가 최신인가? (1-2년 이내)
- [ ] 모순되는 정보는 없는가?
- [ ] 설명이 명확하고 쉬운 말인가?
- [ ] 전문 용어는 설명이 있는가?
